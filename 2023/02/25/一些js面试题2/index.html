Promise

Promise是异步编程的一种解决方案。
从语法上他是一个构造函数。
从功能上他封装一个异步操作：比如fs文件操作，数据库，ajax，定时器。比起传统解决方案-回调函数和时间-更加强大，可以解决回调地狱（回调地狱，外层回调的异步执行结果是内部嵌套回调函数的执行条件）。
他有三种状态，fulfilld（成功），rejected（失败），pending（准备）。

基本用法: Promise构造函数接受哟个函数作为参数，这个函数的两个参数分别是resolve和reject，由JavaScript引擎提供，无需自己部署。
resolve函数的作用是将promise对象的状态从pending变为fulfilld，在异步操作成功时执行，并将异步操作的结果作为参数传递出去；reject函数将状态从pending变为rejected，在异步操作失败是调用，并将代表错误的参数传递出去。(函数并不会终止代码执行)
promise实例生成后，可以调用then指定的resolved状态和rejected状态的函数

Promise.all():

Promise.all()用来将多个promise实例包装成一个新的promise实例。
var p = Promise.all([p1,p2,p3])p1,p2,p3都是Promise实例，如果不是，则会调用Promise.resolve()方法将参数转化伪Promise实例。
p的状态由p1，p2，p3决定（p123都为fulfilled，p的状态才会变成fulfilled，只要p1，p2，p3一个变为rejected，p的状态就是rejected）。

Promise.race()

Promise.race()同样是将多个Promise实例包装成一个新的Promise实例，一组状态谁率先改变，p的状态就跟随改变。

Promise.resolve()

有时候需要将现有对象转为Promsie对象，Promise.resolve()就起这样的作用。他的转化分为以下四种情况：

  1. 参数是一个Promise对象，原封不动返回。
  2. 参数是一个thenable对象，即含有then方法的对象，比如 Promise.resolve方法回家这个对象转为Promise对象，然后立刻执行then方法 上面的代码，obj的then方法执行后，对象p1的状态就变为resolve(),从执行then方法指定的回调。
  3. 对象是个普通对象或者不是对象，Promise.resolve()方法返回一个新的Promise对象，状态为Resolve。
  4. 不带参数，直接返回一个状态为resolve的Promise实例。

Promise.reject()

返回一个状态为rejected的新的Promise实例，他与resolve()不同，rejected()方法的参数会原封不动的作为rejected的理由变为后续的方法的参数。

Generator函数

Generator函数同样是异步编程的解决方案。
从语法上，Generator函数十余个状态机，封装了多个内部状态。
从形似上他是一个函数，不同于普通函数，他是可以暂停执行的，要加*做区分。
整个generator函数就是与各封装的异步任务，异步操作需要在暂停的地方加个yield。
函数返回一个迭代器。

async/await

async/await简单来说就是Generator函数的语法糖，将*换为async，将yield换为await。
他对比传统方式有以下几点改进：

  1. 内置执行器：
    Generator函数的执行必须依靠执行器，async自带执行器，他的执行与普通函数一样
  2. 跟好的语义：
    相比型号和yield，语义清楚，async表示函数中有异步操作，await表示紧跟在后边的表达式需要等待结果。
  3. 更广的适用性：
    co模块规定，yield命令后面只能是Thunk函数或Promise对象，而async函数的await命令后面，可以是Promise对象和原始类型的值（原始类型此时相当于同步）
  4. 返回Promise对象，比Generator函数的返回值是Iterator对象方便，可以用then方法指定下一步操作
    总的来说 async函数就是多个异步操作封装成的一个Promise对象，而await是then的语法糖。
