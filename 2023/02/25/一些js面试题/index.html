一些js面试题目

发表于2023-02-25|更新于2023-03-08
|阅读量:

罗列一些js常见面试题

javaScript

1.js有几种基本数据类型？

基本数据类型有number string boolean undefind null ES6中还引入了一种新的原始类型symbol

2.如何让判断js数据类型？

有typeof和instanceof 或者toString方法

3.typeof和instanceof的区别？

typeof检测数据类型返回值有number，boolean，string，function，object，undefined。
instanceof是检测某个实例的原型链（_proto_）
是否有某个构造的prototype对象

4.如何判断一个变量是不是Array？

-instanceof：arr instanceof Array。
-cunstructor：arr.constructor===Array
-Object.prototype.toString().call()：
Object.prototype.toString.call(a)===’[object Array]’
-Array.isArray(): Array.isArray(arr)

5.==和===的区别

==在进行比较时如果两边数据格式不同，会进行隐式转换
===不会进行隐式转换

6.js事件机制

js是一个事件驱动的语言，无论是点击事件还是定时器，都是在达到某个条件，这些异步任务会加入异步任务队列中，通过事件轮询的机制来执行。

7.事件委托

事件委托原理为冒泡，js的事件流分为捕获-目标-冒泡，自顶向下，在这个过程中，沿途触发节点的同类型事件。0级dom事件只有冒泡触发，且相同事件只能绑定一个，2级dom有捕获阶段触发和冒泡两种，并且可以监听多个相同的事件，并依次执行

8.e.preventDefalue()

e.perventDefalue()用来阻止元素默认事件，比如a标签点击后就会尝试跳转。

9.js中的this是如何工作的

一般情况下，函数中的this是谁调用就指向谁，this是在运行时动态决定的

对于箭头函数 它本身没有this，借助的是外层的非箭头函数中的this，他的this也是变化的，取决于外层函数的this

10.手写apply bind call

call：执行并修改函数内this指向,call(obj,参数1，参数2…)

每个函数都是Function对象(构造函数),每个构造函数都有原形对象(Function.prototype),call就是prototype里的。如果要模仿，就要在原型中添加call。

apply：与call的区别在于 第二个参数是个数组 将函数执行所用的参数集合起来。

bind：bind和不会去执行函数,他会返回一个函数，参数和apply一样

伪数组使用Array的方法

使用的前提是实现了迭代器 以slice为例子
Array.from(obj).slice
Array.prototype.slice.call(obj)
[].slice.call(obj)

new原理

js继承是如何实现的

1–原型链继承 父类的实例作为子类的原型

优点：父类新增的实例和属性子类都能访问
缺点：在创建子类实例时，无法向父类构造传递参数,无法实现多继承
2–利用构造函数继承(伪造对象，经典继承)

优点：解决了子类向父类传参的问题 可以实现多继承(call多个父类)
缺点：只能继承父类构造的属性和方法，原型上的不行。
3–实例继承(原型式继承)？

4–组合式继承，调用父类构造，继承父类属性，通过将父类实例作为子类原型，实现函数复用

缺点：由于调用了两次父类，所以产生了两份实例
优点：函数可以复用,不存在引用属性问题,可以继承属性和方法，并且可以继承原型的属性和方法
5–寄生组合继承,通过寄生的方式来修复组合式继承的不足，完美的实现继承

6–ES6的class，拒绝花里胡哨
通过extends实现继承，比ES5修改原型链等更加清晰易懂

js原型和原型链

所有的原型链的终点都是Object.prototype,
它指向的原型对象依然有原型，不过是null，null没有原型。一句话构造的prototype和它的实例的__proto__指向一个东西

闭包与作用域链

闭包实际就在一个函数内定义一个函数，这个内部定义的的函数 有上层函数环境中变量的引用，并且这个内部函数在全局有引用（return出来这个函数，或者这个函数被定时器引用，亦或是事件函数），那么外部函数环境的变量就不会释放掉。普通函数执行完后内存就释放掉了，他延长了函数环境内变量的生命周期（仅仅是成闭包的那个变量）。但可能带来的是内存泄漏问题。
作用域链就是定义好的函数，作用域一级一级向、下走。闭包就是沿着作用域寻找

let const var

let：他有自己单独的块级作用域，在之前的全局作用域和函数作用域多了块级作用域。
声明的变量只在let命令所在的代码块中生效。
不允许重复声明。
他没有变量提升机制。
有暂时性死区问题 他所在的块一旦声明一个变量，这个变量和代码块绑定，不会受外部影响，总之在代码块内在使用let声明变量之前使用该变量都会报错。
const：声明一个常量，除了不能改变其他和let类似。

js中变量提升

var声明的变量会被提升到作用域顶部，函数式声明的函数会整个提升上去。

js模块化

最开始直接在html写js代码，在全局中定义的函数变量互相冲突污染。然后出现了命名空间和匿名闭包IIFE模式，进行封装提供外部接口供使用，比如jq就是这样。它需要注意依赖顺序

1.首先commonjs规范模块加载是同步的，服务器加载模块从本地读取时间可以忽略不计，但浏览器端不行，由于网络延迟，如果多个js模块之间存在前后依赖，很难保证顺序，所以commonJs奴使用浏览器端。
2.AMD规范适用于浏览器端的异步加载规范，requireJs作为AMD的实现是一个js文件和模块加载器，可以解决浏览器端开发中加载顺序带来的问题
3.CMD规范同样适用于异步加载，SeaJs是CMD的实现
4.ES6模块化的出现，AMD和CMD基本无用了。但目前浏览器还不能完全支持，需要用bable转换。nodeJs也在完善对ES6模块化的支持

IIFE立即执行函数

他一般用来进行变量方法的封装，创建一个私有的函数作用域，避免了向全局添加变量和方法，避免了变量方法冲突，通过闭包延长函数内变量生命周期，同时又不会被外界更改。

匿名函数

匿名函数无需调用立即执行，替换等价的表达式，一般没有必要专门写一个方法 用来做IIFE

箭头函数

箭头函数不需要function关键字，他没有this，他的this指向外层函数环境里的this。

可变对象和不可变对象

可变对象就是可以增删改的对象，不可变对象分为三级。
由弱到强：

  1. Object.isExtensible(obj) //检测是否可扩展，默认true
    设置不可扩展Object.preventExtensions(obj),对象不能新增属性，但可以修改，删除。
  2. Object.isSealed(obj) //检测对象是否密封，默认false
    设置密封Object.seal(obj),对象不可扩展 不能删除，但是可以修改
  3. Object.isFrozen(obj) //检测对象是否冻结，默认为false
    设置对象冻结Object.freeze(obj),对象不能修改，删除，添加。

cookie localStorage sessionStorage

cookie：
cookie是服务端发送给客户端的凭证，因为http/s是无状态无连接的，浏览器每次访问都携带这个数据，就不用反复的验证，比如登陆一个网站退出再回来，不用重新登录。cookie存储上限4kb，浏览器一般允许存储300个，每个站点最多20个。

localStorage和sessionStorage:
他们是H5的新特性。
localStorage：本地存储
优点：
存储量大约在5M
不会过期，可以长期存储数据
不会随着http请求一起发送
在同一个浏览器的不同窗口操作的是同一localStorage
缺点：
不同窗口下数据不能独立，互相干扰。
sessionStorage：会话存储
特点：
存入相同属性，以最后一次复制为准。
如果浏览器一直不关闭，数据一直保存，会导致其他页面中有一些不要的数据那么这些数据(1.占用内存。2.一不小心可能会成为脏数据。)
解决方案：可以使用清空数据，进行清空。
优点：
不同的窗口下的sessionStorage存储独立，不互相干扰。
缺点：
有效期：sessionStorage的有效期只是网页在浏览器打开到关闭这个时间段。

什么是单页应用？如何使其对搜索引擎友好 （未解决）

单页即一个web项目只有一个页面；项目中的内容被分为一个个可复用的组件。
优点：
单页应用会一次性将所有代码请求回来，利用js变化html内容，搭配loaction，histroy可以实现前端路由。
用户体验好，内容改变不用重新加载。
减轻服务器压力，服务器只需要提供数据，而不会把前后端逻辑混为一谈。
组件化思想，代码复用性提升，更加规范。
缺点：
首次加载可能会慢。
导航不可用。
对搜索引擎（SEO）不友好，原因：浏览器抓取内容是通过完整的html文件，而单页通过js来控制视图渲染，（搜索引擎认识html的内容不认识js）所以不能很好的支持搜索。
解决方案：用 #!号代替#号：搜索引擎抓取页面首先要遵循http协议，可是#不是协议内的内容。而实际上也是这样，我们没有见过搜索引擎的搜索结果中，哪一条记录可以快速定位到网页内的某个位置的。解决的方法是用 #!号代替#号，因为谷歌会抓取带有#!的URL。我们可以解决ajax的不被搜索引擎抓取的问题
